# 快速入门指南

从零开始到完成首次网络扫描不到 10 分钟。本指南假设您已经将 NetRecon OS 镜像写入了存储设备。

## 您需要准备的

- 一台已安装 NetRecon OS 的探针设备（参见 [安装](./installation.md)）
- 一条连接到网络的以太网线缆
- 一部手机或电脑，且与探针在同一网络
- NetRecon Scanner 应用（可选，用于移动扫描）

## 第 0-2 分钟：启动探针

1. 插入准备好的 microSD 卡或从内部存储启动
2. 将以太网线缆连接到网络交换机或路由器
3. 开启设备电源
4. 等待绿色状态 LED 变为常亮（大约 60 秒）

## 第 2-5 分钟：完成设置向导

1. 从路由器的 DHCP 表中找到探针的 IP 地址，或查看控制台输出
2. 在浏览器中打开 `http://<探针IP>:8080`
3. 完成以下基本向导步骤：
   - **设置管理员密码** — 选择一个强密码
   - **分配网络接口** — 选择连接到扫描网络的端口
   - **选择扫描模式** — 基本设置选择 "单接口"
   - **配置 Cloudflare Tunnel**（可选） — 启用通过 `https://probe.netreconapp.com` 的远程访问
4. 点击 **完成设置**

## 第 5-7 分钟：验证探针仪表板

1. 向导完成后，导航至 `http://<探针IP>:3000`
2. 使用您创建的管理员凭据登录
3. 验证仪表板显示：
   - 系统健康：CPU、内存、存储使用情况
   - 网络接口：至少一个接口处于"扫描"角色
   - 服务：核心服务应显示绿色状态

## 第 7-10 分钟：运行首次扫描

### 方案 A：从探针仪表板

1. 导航至 **扫描 > 开始扫描**
2. 选择目标子网（从您的扫描接口自动检测）
3. 选择 **快速** 扫描配置
4. 点击 **开始**
5. 在仪表板上实时观察设备出现

### 方案 B：从 NetRecon Scanner 应用

1. 在 Android 设备上打开 NetRecon Scanner 应用
2. 如果您在同一网络，应用会通过 mDNS 检测探针
3. 或者，前往 **设置 > 探针连接** 并输入探针 IP
4. 在主屏幕点击 **扫描**
5. 选择您的网络并点击 **开始扫描**

## 扫描过程

1. **ARP 发现** — 探针发送 ARP 请求以找到子网上所有活跃主机
2. **端口扫描** — 对每个发现的主机扫描开放的 TCP 端口
3. **服务检测** — 探测开放端口以识别运行的服务和版本
4. **设备分析** — 探针结合 MAC 地址 OUI 查询、开放端口和服务横幅来识别设备类型

## 常见问题

**问：扫描发现的设备比预期少，为什么？**
答：确保探针在正确的 VLAN/子网上。防火墙或客户端防火墙可能会阻止 ARP 响应。尝试运行 **标准** 配置而非 **快速** 以获得更全面的发现。

**问：可以扫描多个子网吗？**
答：可以。在探针仪表板的 **设置 > 扫描目标** 中配置其他子网。多子网扫描需要适当的路由或多个网络接口。

**问：扫描需要多长时间？**
答：一个 /24 子网的快速扫描通常在 2 分钟内完成。标准扫描需要 5-10 分钟。深度扫描可能需要 15-30 分钟，具体取决于主机数量和扫描的端口数。
