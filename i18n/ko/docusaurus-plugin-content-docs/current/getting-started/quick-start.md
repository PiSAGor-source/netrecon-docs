# 빠른 시작 가이드

처음부터 첫 번째 네트워크 스캔까지 10분 이내에 완료하세요. 이 가이드는 NetRecon OS 이미지를 저장 장치에 이미 기록했다고 가정합니다.

## 필요한 것

- NetRecon OS가 설치된 Probe 장치 ([설치](./installation.md) 참조)
- 네트워크에 연결된 이더넷 케이블
- 같은 네트워크에 있는 스마트폰 또는 컴퓨터
- NetRecon Scanner 앱 (선택 사항, 모바일 스캐닝용)

## 0-2분: Probe 부팅

1. 준비된 microSD 카드를 삽입하거나 내부 스토리지에서 부팅합니다
2. 이더넷 케이블을 네트워크 스위치 또는 라우터에 연결합니다
3. 장치의 전원을 켭니다
4. 녹색 상태 LED가 점등될 때까지 기다립니다 (약 60초)

## 2-5분: 설정 마법사 완료

1. 라우터의 DHCP 테이블에서 Probe의 IP 주소를 찾거나 콘솔 출력을 확인합니다
2. 브라우저에서 `http://<probe-ip>:8080`을 엽니다
3. 다음 필수 마법사 단계를 완료합니다:
   - **관리자 비밀번호 설정** — 강력한 비밀번호를 선택합니다
   - **네트워크 인터페이스 할당** — 스캔 네트워크에 연결할 포트를 선택합니다
   - **스캔 모드 선택** — 기본 설정으로 "단일 인터페이스"를 선택합니다
   - **Cloudflare Tunnel 구성** (선택 사항) — `https://probe.netreconapp.com`을 통한 원격 접근 활성화
4. **설정 완료**를 클릭합니다

## 5-7분: Probe 대시보드 확인

1. 마법사 완료 후 `http://<probe-ip>:3000`으로 이동합니다
2. 생성한 관리자 자격 증명으로 로그인합니다
3. 대시보드에 다음이 표시되는지 확인합니다:
   - 시스템 상태: CPU, RAM, 스토리지 사용량
   - 네트워크 인터페이스: 최소 하나의 인터페이스가 "스캔" 역할
   - 서비스: 핵심 서비스가 녹색 상태 표시

## 7-10분: 첫 번째 스캔 실행

### 옵션 A: Probe 대시보드에서

1. **스캔 > 스캔 시작**으로 이동합니다
2. 대상 서브넷을 선택합니다 (스캔 인터페이스에서 자동 감지)
3. **빠른** 스캔 프로필을 선택합니다
4. **시작**을 클릭합니다
5. 대시보드에서 실시간으로 장치가 나타나는 것을 확인합니다

### 옵션 B: NetRecon Scanner 앱에서

1. Android 장치에서 NetRecon Scanner 앱을 엽니다
2. 같은 네트워크에 있으면 앱이 mDNS를 통해 Probe를 감지합니다
3. 또는 **설정 > Probe 연결**로 이동하여 Probe IP를 입력합니다
4. 홈 화면에서 **스캔**을 탭합니다
5. 네트워크를 선택하고 **스캔 시작**을 탭합니다

## 스캔 중 일어나는 일

1. **ARP 검색** — Probe가 서브넷의 모든 라이브 호스트를 찾기 위해 ARP 요청을 보냅니다
2. **포트 스캐닝** — 검색된 각 호스트에서 열린 TCP 포트를 스캔합니다
3. **서비스 감지** — 열린 포트를 탐사하여 실행 중인 서비스와 버전을 식별합니다
4. **장치 프로파일링** — Probe가 MAC 주소 OUI 조회, 열린 포트 및 서비스 배너를 결합하여 장치 유형을 식별합니다

## FAQ

**Q: 스캔 결과에서 예상보다 적은 장치가 발견되었습니다. 왜인가요?**
A: Probe가 올바른 VLAN/서브넷에 있는지 확인하세요. 방화벽 또는 클라이언트 측 방화벽이 ARP 응답을 차단할 수 있습니다. 더 철저한 검색을 위해 **빠른** 대신 **표준** 프로필을 사용해 보세요.

**Q: 여러 서브넷을 스캔할 수 있나요?**
A: 네. Probe 대시보드의 **설정 > 스캔 대상**에서 추가 서브넷을 구성하세요. 다중 서브넷 스캐닝에는 적절한 라우팅 또는 여러 네트워크 인터페이스가 필요합니다.

**Q: 스캔은 얼마나 걸리나요?**
A: /24 서브넷의 빠른 스캔은 일반적으로 2분 이내에 완료됩니다. 표준은 5-10분이 걸립니다. 심층 스캔은 호스트 수와 스캔되는 포트에 따라 15-30분이 소요될 수 있습니다.
