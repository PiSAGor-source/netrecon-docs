# البدء السريع مع السحابة

NetRecon Cloud هي أسرع طريقة للبدء. لا إعداد خادم، لا Docker — فقط سجّل حسابًا، وانشر مجسًا، وابدأ اكتشاف شبكتك.

## الخطوة 1: إنشاء حسابك

1. انتقل إلى [app.netreconapp.com](https://app.netreconapp.com) وانقر على **التسجيل**
2. أدخل بريدك الإلكتروني واسم الشركة وكلمة المرور
3. تحقق من عنوان بريدك الإلكتروني
4. سجّل الدخول إلى لوحة تحكم NetRecon

## الخطوة 2: إضافة أول موقع

1. في لوحة التحكم، انتقل إلى **المواقع** في الشريط الجانبي
2. انقر على **إضافة موقع**
3. أدخل اسمًا وعنوانًا للموقع (مثل: "المكتب الرئيسي — إسطنبول")
4. احفظ الموقع

## الخطوة 3: نشر مجس

يحتاج كل موقع إلى مجس واحد على الأقل لاكتشاف الشبكة والمراقبة.

### الخيار أ: NetRecon OS (موصى به)

1. انتقل إلى **المواقع → [موقعك] → المجسات → إضافة مجس**
2. اختر **NetRecon OS** وحمّل الصورة المناسبة لأجهزتك
3. انسخ الصورة إلى بطاقة SD أو SSD باستخدام [balenaEtcher](https://etcher.balena.io/)
4. وصّل المجس بشبكتك عبر Ethernet
5. شغّل الجهاز — سيتصل المجس تلقائيًا بحسابك السحابي عبر Cloudflare Tunnel

### الخيار ب: Docker على خادم موجود

```bash
# سحب وتشغيل حاوية المجس
docker run -d \
  --name netrecon-probe \
  --network host \
  --restart unless-stopped \
  -e ENROLLMENT_TOKEN="your-token-from-dashboard" \
  netrecon/probe:latest
```

احصل على رمز التسجيل من **المواقع → [موقعك] → المجسات → إضافة مجس → Docker**.

### الخيار ج: جهاز افتراضي

1. حمّل ملف OVA من لوحة التحكم
2. استورده في VMware أو Proxmox أو Hyper-V
3. اضبط الجهاز الافتراضي على **الشبكة الجسرية** (مطلوب لفحص الطبقة الثانية)
4. شغّل الجهاز الافتراضي — سيظهر في لوحة التحكم تلقائيًا

## الخطوة 4: بدء الفحص

بمجرد أن يكون المجس متصلاً:

1. انتقل إلى **المواقع → [موقعك] → الأجهزة**
2. انقر على **فحص الآن** أو انتظر الاكتشاف التلقائي (يعمل كل 15 دقيقة)
3. ستظهر الأجهزة المكتشفة في قائمة جرد الأجهزة

## الخطوة 5: تثبيت تطبيق الهاتف المحمول

حمّل **NetRecon Scanner** من Google Play Store لفحص الشبكات أثناء التنقل:

- افحص أي شبكة يتصل بها هاتفك
- تتم مزامنة النتائج تلقائيًا مع لوحة التحكم السحابية
- راجع [نظرة عامة على الماسح](../scanner/overview) للتفاصيل

## ما التالي؟

- **نشر الوكلاء** على الأجهزة الطرفية للحصول على رؤية أعمق → [تثبيت الوكلاء](../agents/overview)
- **إعداد التنبيهات** للأجهزة الجديدة أو الثغرات أو انقطاع الخدمة
- **تكوين التكاملات** مع أدواتك الحالية (LDAP، SIEM، Jira، ServiceNow)
- **دعوة فريقك** عبر **الإعدادات → إدارة الفريق**

## السحابة مقابل الاستضافة الذاتية

| الميزة | السحابة | الاستضافة الذاتية |
|---|---|---|
| إدارة الخادم | تدار بواسطة NetRecon | أنت تدير |
| موقع البيانات | NetRecon Cloud (الاتحاد الأوروبي) | بنيتك التحتية |
| التحديثات | تلقائية | يدوية (docker pull) |
| Cloudflare Tunnel | مضمّن | أنت تقوم بالتكوين |
| التسعير | اشتراك | مفتاح ترخيص |

تحتاج استضافة ذاتية بدلاً من ذلك؟ راجع [دليل التثبيت](../self-hosting/installation).

للمساعدة، تواصل مع [support@netreconapp.com](mailto:support@netreconapp.com).
