# Требования

На этой странице подробно описаны минимальные и рекомендуемые требования для всех компонентов NetRecon.

## Оборудование зонда

### Поддерживаемые платформы

| Устройство | Уровень поддержки | Примечания |
|---|---|---|
| Orange Pi R2S (8 ГБ) | Основной | Два Ethernet-порта, компактный форм-фактор |
| Raspberry Pi 4 (4/8 ГБ) | Основной | Широко доступен, хорошая производительность |
| Raspberry Pi 5 (4/8 ГБ) | Основной | Лучшая производительность ARM |
| x86_64 мини-ПК (Intel N100+) | Основной | Лучшая общая производительность, несколько сетевых интерфейсов |
| Другие ARM64 SBC | Продвинутый | Может потребовать ручной настройки |
| Виртуальные машины (VMware/Proxmox/Hyper-V) | Поддерживается | Требуется мостовая сеть |

### Аппаратные характеристики

| Требование | Минимум | Рекомендуется |
|---|---|---|
| Архитектура | ARM64 или x86_64 | ARM64 четырёхъядерный или x86_64 |
| Ядра ЦПУ | 2 | 4+ |
| ОЗУ | 4 ГБ | 8 ГБ |
| Хранилище | 16 ГБ (eMMC/SD/SSD) | 32 ГБ SSD |
| Ethernet-порты | 1 | 2+ (для режима моста/TAP) |
| USB | Не требуется | USB-A для последовательного консольного адаптера |
| Питание | 5В/3А (SBC) | PoE или разъём питания |

### Рекомендации по хранилищу

- **16 ГБ** достаточно для базового сканирования и мониторинга
- **32 ГБ+** рекомендуется при включении захвата PCAP, журналирования IDS или сканирования уязвимостей
- Файлы PCAP могут быстро расти в загруженных сетях; рассмотрите внешнее хранилище для долгосрочного захвата
- База данных SQLite использует режим WAL для оптимальной производительности записи

## Сетевые требования

### Сетевой доступ зонда

| Направление | Порт | Протокол | Назначение |
|---|---|---|---|
| Зонд -> LAN | ARP | Уровень 2 | Обнаружение хостов |
| Зонд -> LAN | TCP (различные) | Уровень 4 | Сканирование портов |
| Зонд -> LAN | UDP 5353 | mDNS | Обнаружение сервисов |
| Зонд -> Интернет | TCP 443 | HTTPS | Cloudflare Tunnel, обновления |
| LAN -> Зонд | TCP 3000 | HTTPS | Веб-панель управления |
| LAN -> Зонд | TCP 8080 | HTTP | Мастер настройки (только первая загрузка) |

### Рекомендации по файрволу

- Зонду **не требуются входящие порты** из интернета при использовании Cloudflare Tunnel
- Зонду необходим **исходящий HTTPS (443)** для подключения к туннелю и обновлений системы
- Для сканирования локальной сети зонд должен находиться в том же сегменте Layer 2, что и целевые устройства (или использовать SPAN/зеркальный порт)

### Cloudflare Tunnel

Удалённый доступ к зонду обеспечивается через Cloudflare Tunnel. Для этого необходимы:
- Активное подключение к интернету на зонде
- Исходящий TCP 443 (входящие порты не нужны)
- Учётная запись Cloudflare (достаточно бесплатного тарифа)

## Часто задаваемые вопросы

**В: Можно ли запустить зонд на Raspberry Pi 3?**
О: Raspberry Pi 3 имеет только 1 ГБ ОЗУ, что ниже минимального требования. Он может работать для базового сканирования, но не поддерживается.

**В: Нужен ли зонду доступ в интернет?**
О: Доступ в интернет необходим только для Cloudflare Tunnel (удалённый доступ) и обновлений системы. Все функции сканирования работают без интернета.

**В: Можно ли использовать USB Wi-Fi адаптер для сканирования?**
О: Сканирование по Wi-Fi не поддерживается. Зонду необходимо проводное Ethernet-подключение для надёжного и полного обнаружения сети.
