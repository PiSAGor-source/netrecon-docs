# Установка NetRecon OS

Данное руководство проведёт вас через установку NetRecon OS на оборудование вашего зонда. Процесс занимает примерно 15 минут от загрузки до полностью работающего зонда.

## Предварительные требования

- Поддерживаемое аппаратное устройство (см. [Требования](./requirements.md))
- Карта microSD (минимум 16 ГБ, рекомендуется 32 ГБ) или USB-накопитель
- Инструмент для записи образов, такой как [balenaEtcher](https://etcher.balena.io/) или Raspberry Pi Imager
- Компьютер для загрузки и записи образа
- Ethernet-кабель, подключённый к вашей сети

## Шаг 1: Загрузите образ

Загрузите подходящий образ для вашего оборудования из клиентского портала NetRecon:

| Оборудование | Файл образа | Архитектура |
|---|---|---|
| Orange Pi R2S | `netrecon-os-arm64.img.xz` | ARM64 |
| Raspberry Pi 4/5 | `netrecon-os-arm64.img.xz` | ARM64 |
| x86_64 мини-ПК | `netrecon-os-amd64.iso` | x86_64 |

:::tip
Проверьте контрольную сумму загруженного файла со значением, указанным на портале, чтобы убедиться в целостности файла.
:::

## Шаг 2: Запишите образ

### Для устройств ARM64 (Orange Pi, Raspberry Pi)

1. Вставьте карту microSD в компьютер
2. Откройте balenaEtcher
3. Выберите загруженный файл `.img.xz` (извлечение не требуется)
4. Выберите карту microSD в качестве целевого устройства
5. Нажмите **Flash** и дождитесь завершения

### Для устройств x86_64

1. Вставьте USB-накопитель в компьютер
2. Откройте balenaEtcher
3. Выберите загруженный файл `.iso`
4. Выберите USB-накопитель в качестве целевого устройства
5. Нажмите **Flash** и дождитесь завершения
6. Загрузите мини-ПК с USB-накопителя и следуйте инструкциям установщика на экране

## Шаг 3: Первая загрузка

1. Вставьте карту microSD (или внутренний диск для x86_64) в устройство зонда
2. Подключите хотя бы один Ethernet-кабель к вашей сети
3. Включите устройство
4. Подождите примерно 60 секунд для инициализации системы

При первой загрузке зонд получит IP-адрес через DHCP.

## Шаг 4: Запустите мастер настройки

1. С любого устройства в той же сети откройте веб-браузер
2. Перейдите по адресу `http://<probe-ip>:8080`
3. Мастер настройки проведёт вас через начальную конфигурацию

Мастер поможет настроить:
- Учётные данные администратора
- Роли сетевых интерфейсов
- Режим сканирования сети
- Подключение Cloudflare Tunnel
- Параметры безопасности

Подробную документацию по мастеру см. в [Обзоре мастера настройки](../setup-wizard/overview.md).

## Шаг 5: Подключите приложения

После завершения работы мастера:

- **NetRecon Scanner**: может обнаружить зонд через mDNS в локальной сети
- **Admin Connect**: отсканируйте QR-код, отображённый в мастере, или подключитесь через `https://probe.netreconapp.com`

## Аппаратные требования

| Требование | Минимум | Рекомендуется |
|---|---|---|
| ЦПУ | ARM64 или x86_64 | ARM64 четырёхъядерный или x86_64 |
| ОЗУ | 4 ГБ | 8 ГБ |
| Хранилище | 16 ГБ | 32 ГБ |
| Ethernet | 1 порт | 2+ порта |
| Сеть | Доступен DHCP | Предпочтительно статический IP |

## Устранение неполадок

### Не удаётся найти зонд в сети

- Убедитесь, что Ethernet-кабель правильно подключён и светодиод связи активен
- Проверьте таблицу DHCP-аренды вашего маршрутизатора на наличие нового устройства с именем `netrecon`
- Попробуйте подключить монитор и клавиатуру, чтобы увидеть IP-адрес зонда в консоли

### Мастер не загружается

- Убедитесь, что вы обращаетесь к порту 8080: `http://<probe-ip>:8080`
- Сервис мастера запускается примерно через 60 секунд после загрузки
- Проверьте, что ваш компьютер находится в той же сети/VLAN, что и зонд

### Ошибка записи образа

- Попробуйте другую карту microSD; некоторые карты имеют проблемы совместимости
- Загрузите образ заново и проверьте контрольную сумму
- Попробуйте альтернативный инструмент для записи образов

## Часто задаваемые вопросы

**В: Можно ли установить NetRecon OS на виртуальную машину?**
О: Да, ISO-образ x86_64 может быть установлен в VMware, Proxmox или Hyper-V. Выделите минимум 4 ГБ ОЗУ и убедитесь, что ВМ имеет мостовой сетевой адаптер.

**В: Как обновить NetRecon OS после установки?**
О: Обновления доставляются через приложение Admin Connect или через веб-панель зонда в разделе **Настройки > Обновление системы**.

**В: Можно ли использовать Wi-Fi вместо Ethernet?**
О: Зонду необходимо хотя бы одно проводное Ethernet-подключение для надёжного сканирования сети. Wi-Fi не поддерживается в качестве основного интерфейса сканирования.
